# Business Rules — Frontend

## BR-F1: 장바구니 규칙

| ID | 규칙 | 적용 시점 |
|---|---|---|
| BR-F1.1 | 수량은 1 이상이어야 함 (0 이하 시 항목 제거) | 수량 변경 시 |
| BR-F1.2 | 장바구니는 localStorage에 저장 (새로고침 유지) | 모든 변경 시 |
| BR-F1.3 | 주문 성공 시 장바구니 자동 비우기 | 주문 성공 후 |
| BR-F1.4 | 총 금액 = SUM(price * quantity) 실시간 계산 | 모든 변경 시 |
| BR-F1.5 | 장바구니가 비어있으면 주문 버튼 비활성화 | 항상 |

## BR-F2: 주문 규칙

| ID | 규칙 | 적용 시점 |
|---|---|---|
| BR-F2.1 | 주문 확정 중 중복 클릭 방지 (isSubmitting 상태) | 주문 확정 시 |
| BR-F2.2 | 주문 실패 시 장바구니 데이터 유지 | 주문 실패 시 |
| BR-F2.3 | 주문 성공 후 5초 카운트다운 → 메뉴 화면 리다이렉트 | 주문 성공 후 |

## BR-F3: 인증 규칙

| ID | 규칙 | 적용 시점 |
|---|---|---|
| BR-F3.1 | 테이블 자격 증명 localStorage 저장 → 자동 로그인 | 앱 로드 시 |
| BR-F3.2 | 관리자 토큰 localStorage 저장 → 새로고침 유지 | 앱 로드 시 |
| BR-F3.3 | 401 응답 시 토큰 삭제 + 로그인 페이지 리다이렉트 | API 응답 시 |
| BR-F3.4 | 인증되지 않은 사용자는 보호된 라우트 접근 불가 | 라우팅 시 |
