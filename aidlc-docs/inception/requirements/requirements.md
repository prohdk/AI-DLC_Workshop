# 테이블오더 서비스 — 요구사항 정의서

## Intent Analysis

- **User Request**: 디지털 테이블오더 서비스 구축 (고객 주문 + 관리자 모니터링)
- **Request Type**: New Project (Greenfield)
- **Scope Estimate**: System-wide (프론트엔드 + 백엔드 + DB + 실시간 통신)
- **Complexity Estimate**: Complex (다중 사용자 유형, 실시간 SSE, 세션 관리, CRUD + 대시보드)

---

## 기술 스택

| 구분 | 선택 |
|---|---|
| Backend | Python + FastAPI |
| Frontend | React (TypeScript) |
| Database | PostgreSQL |
| 실시간 통신 | Server-Sent Events (SSE) |
| 배포 환경 | 로컬 / Docker Compose |
| 인증 | JWT 토큰 기반 (bcrypt 해싱) |

---

## 설계 결정 사항

| 항목 | 결정 |
|---|---|
| 매장 모델 | 단일 매장 (멀티테넌트 아님) |
| 메뉴 이미지 | 외부 이미지 URL 직접 입력 (업로드 기능 없음) |
| SSE 동시접속 | 다중 관리자 세션 지원 (여러 탭/기기) |
| 테이블 규모 | 중규모 11~30개 |
| 관리자 계정 | 시드 데이터로 DB 직접 삽입 (회원가입 없음) |
| UI 언어 | 한국어 + 영어 (기본 2개 언어) |

## 기능 요구사항

### FR-1: 고객용 기능

#### FR-1.1: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 수행 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지

#### FR-1.2: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지(외부 URL)
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-1.3: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-1.4: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시 → 장바구니 자동 비우기 → 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-1.5: 주문 내역 조회
- 현재 테이블 세션 주문만 표시 (이전 세션 제외)
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 상태(대기중/준비중/완료)
- 매장 이용 완료 처리된 주문 제외

### FR-2: 관리자용 기능

#### FR-2.1: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱, 로그인 시도 제한

#### FR-2.2: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트 (2초 이내)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 다중 관리자 세션 동시 접속 지원

#### FR-2.3: 테이블 관리
- **테이블 초기 설정**: 테이블 번호 및 비밀번호 설정, 세션 생성
- **주문 삭제**: 확인 팝업 → 주문 즉시 삭제 → 총 주문액 재계산
- **테이블 세션 처리**: 이용 완료 시 주문 내역을 과거 이력으로 이동, 현재 주문 목록 및 총 주문액 리셋
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-2.4: 메뉴 관리
- 메뉴 CRUD (조회/등록/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

## 비기능 요구사항

### NFR-1: 성능
- SSE 주문 알림 2초 이내 전달
- 테이블 11~30개 규모 동시 처리
- 다중 관리자 SSE 세션 동시 지원

### NFR-2: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 한국어 + 영어 기본 지원
- 직관적 카드 기반 레이아웃

### NFR-3: 신뢰성
- 장바구니 로컬 저장 (새로고침 시 유지)
- 주문 실패 시 장바구니 데이터 보존
- 세션 만료 시 자동 로그아웃

### NFR-4: 배포
- Docker Compose 기반 로컬 배포
- 단일 매장 운영

## 제외 기능 (Constraints)

다음 기능은 MVP 범위에서 **명시적으로 제외**됩니다:

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 업로드/리사이징/최적화
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어 시스템)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도, 번역 API)

## 데이터 모델 개요

### 핵심 엔티티
- **Store**: 매장 정보 (단일 매장)
- **Admin**: 관리자 계정 (시드 데이터)
- **Table**: 테이블 정보 (번호, 비밀번호, 현재 세션)
- **TableSession**: 테이블 세션 (시작~이용완료)
- **Category**: 메뉴 카테고리
- **MenuItem**: 메뉴 항목 (이름, 가격, 설명, 이미지 URL, 노출 순서)
- **Order**: 주문 (테이블, 세션, 상태, 총 금액)
- **OrderItem**: 주문 항목 (메뉴, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (세션 종료 시 이동)
